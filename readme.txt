enhanced_long_term_selector.py和long_term_selector.py的核心区别是什么


核心区别就三点：

评分维度不同
long_term_selector.py：主要是技术面 + 资金面（趋势、动量、量能、ADX、波动、乖离、资金流）。
enhanced_long_term_selector.py：在上述基础上加了基本面与估值（ROE、利润增长、股息率、PE/PEG）和 DMI 信号融合。
打分框架不同
long_term_selector.py：总分 100，技术指标权重更集中。
enhanced_long_term_selector.py：先按扩展维度打分（内部 max_score=130），再归一化到 100，并用 optimize_signal_trigger 做信号共振决策。
依赖与输出不同
long_term_selector.py：依赖 advanced_indicators.py，输出偏技术交易建议。
enhanced_long_term_selector.py：额外依赖 advanced_long_term_indicators.py + fundamental_data.py，输出会包含基本面/估值解释，选股更偏“中长线质量+估值”。

如何单独想算002594这个股票的评分
在你这个目录运行：

cd C:\Users\wzk1311\.openclaw\workspace\skills\a-stock-monitor\scripts
python -c "from short_term_selector import ShortTermSelector; s=ShortTermSelector(verbose=False); r=s.analyze_single_stock('002594'); print(r if r else 'no result'); s.close()"
如果你只想看核心结果（评分/评级）：

python -c "from short_term_selector import ShortTermSelector; s=ShortTermSelector(verbose=False); r=s.analyze_single_stock('002594'); print({'code':r['code'],'name':r['name'],'score':r['score'],'rating':r['rating']} if r else 'no result'); s.close()"
注意：

这个函数会用本地 stock_cache.db 里的 get_stock() 做基础信息，所以先跑过 update_all_market_data.py 更稳。
资金流如果缓存里没有，会少一部分分值，但仍可算出结果。
